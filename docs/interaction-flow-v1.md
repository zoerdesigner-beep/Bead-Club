# 拼豆工具网站 V1 交互流程稿

## 1. 目标

本文件补充 `docs/product-requirements-v1.md`，用于说明 V1 的页面状态、操作路径和关键交互细节。它不涉及具体代码实现，重点帮助后续进行原型设计和开发拆解。

## 2. 页面清单

| 页面 | 主要目标 | 核心操作 |
| --- | --- | --- |
| 首页 | 解释产品价值，引导上传 | 上传图片生成图纸 |
| 工具页 | 完成生成和下载闭环 | 上传、设置、生成、下载 |
| 说明页 | 解释色板、效果和风险 | 阅读说明、返回工具 |

## 3. 首页交互

### 3.1 首屏

展示内容：
- 标题：把图片转换成拼豆图纸
- 副标题：上传图片，自动生成基于 MARD 291 色板的网格图纸、色号和用料统计。
- 主按钮：上传图片生成图纸
- 次级入口：查看使用说明

交互规则：
- 点击主按钮后直接打开文件选择器。
- 上传成功后进入工具页，并带入已上传图片。
- 点击“查看使用说明”进入说明页。

### 3.2 示例区

展示内容：
- 原图示例。
- 转换后的拼豆网格图。
- 示例参数：29 x 29、32 色、总豆数。

交互规则：
- 示例区只用于展示，不提供编辑能力。
- 可提供“试试这个示例”按钮，V1 可选。

## 4. 工具页交互

### 4.1 初始状态

当用户未上传图片时：
- 页面中心显示上传卡片。
- 文案：上传一张图片，生成你的第一张拼豆图纸。
- 辅助说明：支持 JPG、PNG、WebP，最大 10MB。

可用操作：
- 点击上传。
- 拖拽图片上传。

不可用操作：
- 生成图纸。
- 下载 PNG。

### 4.2 上传成功状态

上传成功后展示：
- 原图预览。
- 参数设置区。
- 默认参数：
  - 宽度：29 格。
  - 最大颜色数：32 色。
  - 透明处理：保留透明。
- 生成按钮。

交互规则：
- 用户可以直接点击生成。
- 用户也可以先调整参数。
- 再次上传图片会替换当前图片并清空旧结果。

### 4.3 参数设置

#### 宽度格数

控件建议：
- 快捷按钮：14、29、58、87、116。
- 自定义数字输入框。

交互规则：
- 选择快捷按钮时，同步更新输入框。
- 输入范围为 8 到 200。
- 输入非法时展示错误，并禁用生成按钮。
- 宽度变化后，实时显示预计高度和总豆数。

#### 最大颜色数

控件建议：
- 单选按钮或下拉选择。
- 选项：8、16、24、32、48、64、96、无限制。

交互规则：
- 默认选择 32。
- 参数变化后，如果已有生成结果，展示“参数已变化，请重新生成”。

#### 透明背景

控件建议：
- 单选项：
  - 保留透明。
  - 填充白色背景。

交互规则：
- 仅在图片含透明通道时突出展示。
- 默认保留透明。
- 切换后需要重新生成。

### 4.4 生成中状态

展示内容：
- 加载提示：正在生成图纸，图片越大可能需要越久。
- 禁用生成按钮。
- 禁用下载按钮。

交互规则：
- 用户仍可查看原图和参数。
- 生成完成后进入预览状态。
- 生成失败后进入错误状态，保留参数。

### 4.5 生成成功状态

展示内容：
- 图纸预览。
- 参数摘要：宽度 x 高度、总豆数、实际颜色数。
- 用料统计表。
- 下载 PNG 按钮。

可用操作：
- 下载 PNG。
- 调整参数并重新生成。
- 重置。
- 重新上传。

提示规则：
- 实际颜色数大于 48：提示“颜色较多，备料复杂度较高”。
- 总豆数大于 2500：提示“图纸较大，制作时间较长”。
- 宽度大于 58：提示“后续可使用分板打印功能，目前 PNG 可能较大”。

### 4.6 生成失败状态

展示内容：
- 错误文案。
- 失败原因。
- 重新生成按钮。

常见失败原因：
- 图片无法解析。
- 参数不合法。
- 图纸尺寸过大导致浏览器处理失败。

交互规则：
- 保留用户上传图片。
- 保留参数。
- 用户可以降低宽度或颜色数量后重新生成。

## 5. 图纸预览规则

### 5.1 预览内容

图纸预览应展示：
- 网格。
- 每格颜色。
- 色号或短编号。
- 透明区域。

### 5.2 色号显示

规则：
- 小尺寸图纸优先直接显示完整 MARD 色号。
- 大尺寸图纸如果格子过小，可在页面预览中隐藏文字，但下载图应尽量保留色号或编号。
- 如果使用短编号，图例必须展示短编号和 MARD 色号的对应关系。

### 5.3 文字颜色

规则：
- 浅色背景使用深色文字。
- 深色背景使用浅色文字。
- 文字与背景对比不足时优先保证图例可读。

## 6. 用料统计交互

展示字段：
- 颜色块。
- MARD 色号。
- HEX。
- 实际颗数。
- 建议准备颗数。

排序：
- 默认按实际颗数从多到少排序。

辅助说明：
- 建议准备颗数按实际颗数增加约 5% 计算。
- 透明格不计入总豆数。

## 7. 下载 PNG 交互

点击“下载 PNG”后：
- 生成一张完整导出图。
- 文件名为 `mard-pattern-{width}x{height}.png`。
- 下载图包含图纸和图例。

导出图内容：
- 标题：MARD 拼豆图纸。
- 尺寸：例如 29 x 42。
- 总豆数。
- 实际颜色数。
- 拼豆网格。
- 用料图例。
- 色板说明：MARD 291 色值为社区整理参考，实物可能存在色差。

失败处理：
- 如果下载生成失败，提示“下载失败，请稍后重试或降低图纸尺寸”。

## 8. 说明页内容结构

### 8.1 MARD 291 色板说明

需要说明：
- 当前 V1 仅支持 MARD 291。
- 色值来自社区整理数据。
- 屏幕显示和实物豆子可能存在偏差。

### 8.2 自动转图效果说明

需要说明：
- 工具会把图片缩小成指定格数。
- 每个格子会映射到最接近的 MARD 色号。
- 颜色越少越容易制作，但细节会减少。
- 颜色越多越接近原图，但备料更复杂。

### 8.3 推荐图片

推荐：
- 像素风图片。
- 卡通头像。
- 图标。
- 主体清晰、背景简单的图片。

不推荐：
- 复杂人像。
- 风景照片。
- 大量渐变和阴影。
- 小字很多的图片。

### 8.4 版权说明

文案建议：
请仅上传你拥有使用权的图片。本工具仅用于生成个人拼豆图纸，不会主动保存或发布你的图片。

## 9. V1 完成定义

V1 完成的判断标准：
- 用户可以从首页进入上传。
- 用户可以在工具页完成图片上传。
- 用户可以设置宽度和最大颜色数。
- 用户可以生成 MARD 291 图纸。
- 用户可以看到图纸预览和用料统计。
- 用户可以下载包含图纸和图例的 PNG。
- 页面包含基本说明和免责声明。
